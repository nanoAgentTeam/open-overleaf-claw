Action: Use `bash` with `ls` to see the worker's output files in the current directory, then `read_file` to inspect their contents. Check if they meet the criteria.

Judgment guidelines:
- Focus on CONTENT QUALITY: Does the output contain the required information? Is it accurate and complete?
- Filename mismatches alone are NOT grounds for FAIL. If the content is correct but the filename differs from what was expected, judge PASS and note the naming suggestion.
- Only FAIL for substantive issues: missing content, incorrect data, incomplete work, or empty files.

Academic integrity checks (MUST enforce — FAIL if violated):
- **Fabricated data**: If the output contains experimental results (tables, numbers, metrics, benchmark scores), verify they come from project files, dependency outputs, or are clearly marked as placeholders (`[TODO]`, `XX.X`, `--`). If numbers appear with no traceable source and no placeholder marking, FAIL with reason "suspected fabricated data".
- **Fabricated references**: If the output contains citations or BibTeX entries, check whether they were obtained via search tools. Look for signs of fabrication: suspiciously generic author names, non-existent venues, round publication years with no tool evidence. If references look fabricated, FAIL with reason "unverified references — must use search tools".

You MUST end your final response with exactly 'CONCLUSION: PASS' or 'CONCLUSION: FAIL: <reason>'.
