# 项目隔离与安全

## 角色权限隔离

系统通过角色区分不同 Agent 的权限：

- **Assistant（主 Agent）**：对 core 有完整的读写权限
- **Worker（SubAgent）**：只能读取 core，写入被重定向到自己的 overlay 目录
- **GitAgent**：专用的 Git 操作权限，仅在 `/git` 模式下激活

## 路径安全

- 所有文件操作都经过路径校验，`..` 和绝对路径被拒绝，防止沙箱逃逸
- Bash 命令的工作目录锁定在 core，无法访问外部文件

## 写入保护

- SubAgent 的写入不会直接影响 core，必须经过 merge 流程
- 截断的写入操作会被拒绝，防止文件损坏
- 每次写入都被追踪，turn 结束时统一 auto commit

## 项目文件如何被修改

### 直接修改（主 Agent）

主 Agent 直接写入 core 目录。每次写入会被追踪，turn 结束时自动 git commit。

### 间接修改（SubAgent）

SubAgent 写入自己的 overlay 目录（隔离），完成后通过 merge 机制合并到 core 或 `_subagent_results/`。

### 上下文安全

- Agent 循环有死锁检测：如果同一个工具调用重复 3 次以上，会触发 Meta-Diagnosis（独立的 LLM 分析死锁原因并介入）
- 上下文溢出有三级防护：70% 阈值时主动压缩、安全限制时强制摘要、失败后恢复重试
